# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

**DO NOT** open public issues for security vulnerabilities.

Instead, email: security@aibradaa.com

We'll respond within 48 hours.

### What to Include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

### What to Expect

1. **Acknowledgment** within 48 hours
2. **Assessment** within 7 days
3. **Fix** released based on severity (1-30 days)
4. **Credit** in release notes (if desired)

## Security Measures

- HTTPS/TLS encryption
- JWT authentication
- PDPA compliance
- Rate limiting
- CSP headers
- Input validation
- XSS protection
- CSRF tokens
- Secure password hashing (bcrypt)

## Scope

**In Scope:**
- Authentication bypass
- Data leaks
- XSS/CSRF
- SQL injection (if applicable)
- API vulnerabilities

**Out of Scope:**
- Social engineering
- DoS attacks
- Issues requiring physical access
- Third-party service vulnerabilities

Thank you for helping keep AI Bradaa secure!
